# ОБНОВЛЕННЫЙ АРХИТЕКТУРНЫЙ АУДИТ И CODE REVIEW

Главный девиз текущего этапа разработки, который должен быть выжжен в коде:

> **«Не делай VM. Нарезай константу на жесткие параметры (маски/сдвиги), а саму мясорубку пиши в Rust монолитным branchless-кодом.»**
> 
> 

### В чем фундаментальная суть этого правила?

Если в декодере есть цикл, который читает опкоды и делает `match opcode { ... }`, процессор (CPU) начинает ветвиться на каждом шаге. На случайных или высокоэнтропийных данных блок предсказания переходов (Branch Predictor) CPU сходит с ума, конвейер сбрасывается, скорость падает до нуля.

Вместо виртуальной машины мы строим **инвариантную математическую трубу** (конвейер). Данные летят сквозь неё прямолинейно. Константа `K` — это не «программа» для VM, это **динамический слепок геометрии пространства**. Биты `K` просто нарезаются на параметры, которые вставляются прямо в битовые операции (XOR, циклические сдвиги, умножения в кольце вычетов), не меняя маршрут движения данных.

## 1. Два главных исправления (Где были косяки)

### Исправление I: Метагенератор — это НЕ брутфорс

Никакого перебора 64 вариантов или слепого угадывания производящих функций. Метагенератор работает как **аналитический сканер**:

1. Он применяет **FWHT (Быстрое преобразование Уолша-Адамара)** к блоку данных.
2. Спектральный анализ напрямую выявляет линейные и нелинейные инварианты (резонансы топологии).
3. Параметры трансформации (какие оси разворачивать, какие маски фаз применять) **вычисляются аналитически** на основе пиков спектра, а не угадываются.

### Исправление II: Переменная длина `K` (Согласно PLAN.md v4.1)

Константа `K` не привязана к типу `u64`. Это битовый поток (`bit-packed stream`) переменной длины.

* Если топология блока простая (спектр чистый) — `K` может занимать всего 12 бит.
* Если блок сложный — Метагенератор упаковывает в `K` больше параметров.
* **Ограничитель сверху — только экономика сжатия**: `|K| + |Seed| + |V| + overhead < |Block|` (и жесткая 10%-я гильотина на первом проходе).

## 2. Логические косяки в текущей кодовой базе (`src/`)

### Косяк А: Наличие `ProgramIR` и `StageOpcode` (Файлы `key.rs`, `base.rs`)

В текущем коде всё ещё видны структуры абстрактного синтаксического дерева программы.

* **Что не так:** План `v4.1` требует избавиться от этого. Нам не нужно парсить дерево инструкций во время декодирования.
* **Как исправить:** Убрать `enum StageOpcode`. Вместо этого `Sig` (Topology Signature) блока однозначно определяет, из каких именно полей состоит `K` для данного конкретного шага компиляции. Мы просто нарезаем биты потока `K` по маске, заданной в `Sig`.

### Косяк Б: Массивы дельт и исключений (`crumbs` в `trajectory.rs`)

Наличие векторов ошибок или дельт (ULEB128) указывает на то, что математический оператор не биективен (необратим без подсказок). На белом шуме объём этих «подсказок» превысит размер исходных данных.

* **Как исправить:** Оператор `U_K` обязан быть строго обратимым. Мы сохраняем в архив только `K`, `Seed` и `V` (parity-stream). Если они не дают сжатия — блок бракуется и уходит на следующий `Pass` (уровень рекурсии слоёв), без попыток сохранить «карту ошибок».

## 3. Как должен выглядеть правильный код

Посмотрим, как реализовать **переменную длину константы `K`** и **branchless-конвейер** без создания виртуальной машины.

Вместо интерпретатора мы пишем монолитную структуру, где `Sig` указывает, как распилить биты `K`.

```
//================================================
// FILE: src/domain/kernel/reversible.rs
//================================================

pub struct VariableLengthPipeline;

/// Топологическая сигнатура однозначно определяет, сколько бит 
/// выделено на каждый параметр в константе K для текущего блока.
pub struct TopologySig {
    pub bit_width_mask: u32,
    pub bit_width_shift_a: u32,
    pub bit_width_shift_b: u32,
    pub bit_width_mult: u32,
}

impl VariableLengthPipeline {
    /// Forward transform: Прямолинейный монолитный конвейер.
    /// ZERO branches. Скорость исполнения максимальна.
    #[inline(always)]
    pub fn forward(data: u64, k_stream: &[u8], sig: &TopologySig) -> u64 {
        // --- 1. LANGUAGE-AGNOSTIC EXTRACTION LAYER ---
        // Извлекаем параметры из K переменной длины на основе Сигнатуры Топологии.
        // Никаких циклов и match, только чистые битовые маски из потока.
        let mut bit_offset = 0;
        
        let phase_mask = read_bits(k_stream, &mut bit_offset, sig.bit_width_mask);
        let shift_alpha = read_bits(k_stream, &mut bit_offset, sig.bit_width_shift_a) as u32;
        let shift_beta = read_bits(k_stream, &mut bit_offset, sig.bit_width_shift_b) as u32;
        let multiplier = read_bits(k_stream, &mut bit_offset, sig.bit_width_mult) | 1; // Гарантируем нечетность

        // --- 2. BRANCHLESS MEAT GRINDER ---
        // Слои трансформации идут один за другим жестко. Изменение K меняет 
        // поведение слоев (силу сдвига, геометрию маски), но не порядок выполнения.
        let mut state = data ^ phase_mask;
        
        // Циклические сдвиги (компилируются процессором в аппаратные инструкции ROL/ROR)
        state = state.rotate_left(shift_alpha ^ 1) ^ state.rotate_right(shift_beta % 64);
        
        // Модульное умножение (сжатие/растяжение пространства в кольце Z/2^64Z)
        state = state.wrapping_mul(multiplier);

        state
    }

    /// Inverse transform: Идеальное зеркакальное отражение.
    #[inline(always)]
    pub fn inverse(compressed_state: u64, k_stream: &[u8], sig: &TopologySig) -> u64 {
        let mut bit_offset = 0;
        
        let phase_mask = read_bits(k_stream, &mut bit_offset, sig.bit_width_mask);
        let shift_alpha = read_bits(k_stream, &mut bit_offset, sig.bit_width_shift_a) as u32;
        let shift_beta = read_bits(k_stream, &mut bit_offset, sig.bit_width_shift_b) as u32;
        let multiplier = read_bits(k_stream, &mut bit_offset, sig.bit_width_mult) | 1;

        // Вычисляем модульное обратное для нечетного числа (в кольце 2^64) без ветвлений
        let modular_inverse = Self::compute_modular_inverse_64(multiplier);

        // Разворачиваем конвейер строго в обратном порядке
        let mut state = compressed_state.wrapping_mul(modular_inverse);
        state = state.rotate_right(shift_alpha ^ 1) ^ state.rotate_left(shift_beta % 64);
        state ^= phase_mask;

        state
    }

    #[inline(always)]
    fn compute_modular_inverse_64(n: u64) -> u64 {
        // 5 шагов Ньютона-Рафсона — абсолютно branchless решение
        let mut x = n.wrapping_mul(2).wrapping_sub(n.wrapping_mul(n));
        x = x.wrapping_mul(2u64.wrapping_sub(n.wrapping_mul(x)));
        x = x.wrapping_mul(2u64.wrapping_sub(n.wrapping_mul(x)));
        x = x.wrapping_mul(2u64.wrapping_sub(n.wrapping_mul(x)));
        x = x.wrapping_mul(2u64.wrapping_sub(n.wrapping_mul(x)));
        x
    }
}

/// Утилита чтения произвольного количества бит из байтового потока (zero-copy концепт)
#[inline(always)]
fn read_bits(stream: &[u8], offset: &mut usize, width: u32) -> u64 {
    if width == 0 { return 0; }
    // Быстрое чтение битового окна (в реальном коде заменить на эффективный BitReader)
    let byte_index = *offset / 8;
    let bit_shift = *offset % 8;
    
    let mut val = 0u64;
    for i in 0..8 {
        if byte_index + i < stream.len() {
            val |= (stream[byte_index + i] as u64) << (i * 8);
        }
    }
    
    let mask = (1u64 << width).wrapping_sub(1);
    let result = (val >> bit_shift) & mask;
    *offset += width as usize;
    result
}

```

## 4. Чек-лист действий для выпрямления проекта

1. **Модуль `spectral.rs`:** Настрой FWHT так, чтобы он возвращал аналитически посчитанные коэффициенты. Убери любой намёк на итеративный перебор или брутфорс параметров. Нашёл пик спектра — превратил его координату в биты маски `K`.
2. **Модуль `key.rs` и `base.rs`:** Удалить `ProgramIR` и генерацию байт-кода команд. `K` теперь — чистый вектор бит (`BitVec` / `Vec<u8>`), структура которого диктуется сигнатурой `Sig`.
3. **Модуль `codec.rs`:** Реализовать адаптивную проверку размера: как только суммарная длина считанных бит `K + Seed + V` выходит за рамки лимита (`<= 10%` от размера текущего блока) — мгновенно прерывать обработку (Strict Fail) и отдавать блок на разбиение в следующий `Pass`.

Архитектурный тупик устранен. Теперь это гибкая система с переменной длиной метаданных, работающая на аппаратной скорости CPU. Переноси эту логику в ядро.
