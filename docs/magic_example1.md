
## 1. Берём одну универсальную константу

```text
X = 0x6996057931E
```

Разбираем её твоим способом:

```text
part_3 = X & ((0x1 << 0x4) - 0x1)
part_2 = (X >> 0x4) & ((0x1 << 0x10) - 0x1)
part_1 = (X >> 0x14) & ((0x1 << 0x18) - 0x1)
```

Получается:

```text
part_1 = 0x699605
part_2 = 0x7931
part_3 = 0xE
```

Смысл:

```text
part_1 = Hadamard pack
part_2 = канальный цикл
part_3 = clean-mask
```

---

## 2. Из `part_1` получаем Hadamard

```text
part_1 = 0x699605
```

Разбираем:

```text
K_PARITY = part_1 >> 8
K_PARITY = 0x6996

LOG_N = part_1 & 0xFF
LOG_N = 5

N = 1 << LOG_N
N = 32
```

То есть Hadamard работает на 32 позициях.

---

## 3. Из `part_2` получаем каналы

```text
part_2 = 0x7931
```

Достаём 4 nibble:

```text
m0 = (part_2 >> 0x0)  & 0xF = 1
m1 = (part_2 >> 0x4)  & 0xF = 3
m2 = (part_2 >> 0x8)  & 0xF = 9
m3 = (part_2 >> 0xC)  & 0xF = 7
```

То есть:

```text
[1, 3, 9, 7]
```

---

## 4. Из `part_3` получаем clean

```text
part_3 = 0xE
```

В двоичном виде:

```text
0xE = 1110₂
```

Значит:

```text
keep0 = (part_3 >> 0) & 1 = 0
keep1 = (part_3 >> 1) & 1 = 1
keep2 = (part_3 >> 2) & 1 = 1
keep3 = (part_3 >> 3) & 1 = 1
```

То есть канал `m0 = 1` удаляем. Это и есть очистка самопопадания `p`.

---

# Теперь число 29

Вход:

```text
p = 29
```

Считаем четыре события:

```text
x0 = p * m0 = 29 * 1 = 29
x1 = p * m1 = 29 * 3 = 87
x2 = p * m2 = 29 * 9 = 261
x3 = p * m3 = 29 * 7 = 203
```

---

## 5. Деление на 10 без обычного деления

Используем magic division by 10:

```text
DIV10 = 0xCCCCCCCD
```

Формула:

```text
q = (x * DIV10) >> 35
d = x - ((q << 3) + (q << 1))
```

Потому что:

```text
(q << 3) + (q << 1) = 8q + 2q = 10q
```

---

## 6. Канал 0: `x0 = 29`

```text
q0 = (29 * 0xCCCCCCCD) >> 35 = 2
d0 = 29 - ((2 << 3) + (2 << 1))
d0 = 29 - 20 = 9
keep0 = 0
```

Битовая маска события:

```text
B0 = keep0 << q0
B0 = 0 << 2
B0 = 0x00000000
```

То есть самопопадание удалено.

# 10. Схлопываем события в очищенную битовую маску

```text
E29_BITS = B0 | B1 | B2 | B3
```

Подставляем:

```text
E29_BITS =
0x00000000
| 0x00000100
| 0x04000000
| 0x00100000
```

Получаем:

```text
E29_BITS = 0x04100100
```

Это очищенная битовая маска присутствия специальных символов для 29.

То есть позиции:

```text
8
20
26
```

А позиция `2` удалена, потому что это самопопадание `29`.

# 11. Теперь Hadamard-суперпозиция через тот же `part_1`

Нам нужны Hadamard-маски для позиций:

```text
8, 20, 26
```

Из `LOG_N = 5` разворачиваются базовые паттерны Hadamard:

```text
H0 = 0xAAAAAAAA
H1 = 0xCCCCCCCC
H2 = 0xF0F0F0F0
H3 = 0xFF00FF00
H4 = 0xFFFF0000
```

Они не number-specific. Они выводятся из `LOG_N = 5`.

Для позиции `8`:

```text
8 = 01000₂
```

Значит берём `H3`:

```text
S8 = 0xFF00FF00
```

Для позиции `20`:

```text
20 = 10100₂ = 16 + 4
```

Значит:

```text
S20 = H4 ^ H2
S20 = 0xFFFF0000 ^ 0xF0F0F0F0
S20 = 0x0F0FF0F0
```

Для позиции `26`:

```text
26 = 11010₂ = 16 + 8 + 2
```

Значит:

```text
S26 = H4 ^ H3 ^ H1
S26 = 0xFFFF0000 ^ 0xFF00FF00 ^ 0xCCCCCCCC
S26 = 0xCC3333CC
```

---

# 12. Суперпозиция для очищенной маски 29

```text
SUPER_29 =
S8,
S20,
S26
```

То есть набор битовых Hadamard-масок:

```text
S8  = 0xFF00FF00
S20 = 0x0F0FF0F0
S26 = 0xCC3333CC
```

Это и есть разложение очищенной маски 29 в Hadamard-суперпозицию.

# 13. Как этой же константой схлопнуть обратно

Чтобы проверить позицию `8`, берём её Hadamard-маску:

```text
S8 = 0xFF00FF00
```

Сравниваем с каждым элементом суперпозиции через XOR.

Формула элементарная:

```text
match = 32 - (popcnt(A ^ B) << 1)
```

Если маски одинаковые:

```text
A ^ B = 0
popcnt(0) = 0
match = 32
```

Если разные ортогональные Hadamard-маски:

```text
popcnt(A ^ B) = 16
match = 32 - 32 = 0
```

Для позиции `8`:

```text
match(S8, S8)  = 32
match(S8, S20) = 0
match(S8, S26) = 0
```

Сумма:

```text
32 + 0 + 0 = 32
```

Нормировка:

```text
32 >> LOG_N = 32 >> 5 = 1
```

Значит бит позиции `8` есть.

---

## Позиция `20`

```text
match(S20, S8)  = 0
match(S20, S20) = 32
match(S20, S26) = 0
```

```text
32 >> 5 = 1
```

Бит позиции `20` есть.

---

## Позиция `26`

```text
match(S26, S8)  = 0
match(S26, S20) = 0
match(S26, S26) = 32
```

```text
32 >> 5 = 1
```

Бит позиции `26` есть.

---

## Позиция `2`

Для позиции `2`:

```text
2 = 00010₂
```

Hadamard-маска:

```text
S2 = H1 = 0xCCCCCCCC
```

Проверяем:

```text
match(S2, S8)  = 0
match(S2, S20) = 0
match(S2, S26) = 0
```

Сумма:

```text
0
```

Значит бит позиции `2` отсутствует. То есть самопопадание очищено.

# 14. Схлопнутая маска обратно

```text
BACK =
(1 << 8)
| (1 << 20)
| (1 << 26)
```

```text
BACK = 0x04100100
```

То есть мы получили обратно:

```text
E29_BITS = 0x04100100
```

# 15. Возвращаем hex-символы

Мы помним `d1,d2,d3`:

```text
позиция 8:  d1 = 7 → hex d
позиция 26: d2 = 1 → hex 7
позиция 20: d3 = 3 → hex b
```

Значит очищенная маска:

```text
позиция 8  = d
позиция 20 = b
позиция 26 = 7
остальное = f
```

Строка:

```text
ffffffffdfffffffffffbfffff7ff
```

Если не очищать канал `m=1`, то добавилась бы позиция:

```text
позиция 2 = e
```

И полная маска была бы:

```text
ffefffffdfffffffffffbfffff7ff
```

## Коротко

Одна универсальная константа:

```text
X = 0x6996057931E
```

даёт:

```text
part_1 = 0x699605 → Hadamard H32
part_2 = 0x7931   → каналы [1,3,9,7]
part_3 = 0xE      → удалить самопопадание
```

Для `p = 29`:

```text
B0 = 0x00000000
B1 = 0x00000100
B2 = 0x04000000
B3 = 0x00100000
```

Схлопнутая очищенная битовая маска:

```text
E29_BITS = 0x04100100
```

Hadamard-суперпозиция:

```text
S8  = 0xFF00FF00
S20 = 0x0F0FF0F0
S26 = 0xCC3333CC
```

Схлопывание той же Hadamard-частью обратно:

```text
BACK = 0x04100100
```

И hex-строка очищенной маски:

```text
ffffffffdfffffffffffbfffff7ff
```
